# Contributing

Thank you for your interest in contributing to BrandTrace.

BrandTrace is built around trust, neutrality, and source-based documentation. Contributions should strengthen transparency without turning the project into a political platform, boycott tool, or place for unsupported accusations.

## Contribution Principles

- Prioritize trust over speed.
- Prioritize facts over opinions.
- Prioritize documentation over assumptions.
- Favor clean architecture over shortcuts.
- Keep verified information separate from unverified submissions.
- Use neutral language.

## Acceptable Contributions

Contributions may include:

- Documentation improvements.
- Source research workflows.
- Database model proposals.
- Verification process improvements.
- Accessibility improvements.
- Future application, API, or website code after the foundation phase.

## Evidence Standards

When contributing factual ownership information, include reliable sources whenever possible. Do not present unsupported claims as fact.

## Community Submissions

Community submissions are welcome, but they must be reviewed before becoming verified information. A submission should include:

- The claim being made.
- The product, brand, or company involved.
- Supporting sources.
- Any known uncertainty or conflicting evidence.

## Tone and Conduct

Use respectful, neutral, and careful language. BrandTrace documents ownership information so consumers can decide for themselves.

## Before Building Features

Before implementing any feature, ask:

1. Does this increase transparency?
2. Does this improve trust?
3. Does this empower consumers?
4. Can this information be verified?
5. Will this still make sense five years from now?
